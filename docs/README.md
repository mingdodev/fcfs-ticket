## 핵심 요약

- **목표**: 1,000장 한정 판매 티켓의 선착순 예매 서비스를 5,000명 동시 요청으로 테스트
- **절대 원칙**: Oversell(초과 판매) 금지 - 데이터 정합성 > 성능
- **환경**: 단일 NCP 최저 사양 서버에서 Docker로 Spring Boot + MySQL 실행
- **목표 TPS**: 100~200 TPS 달성 (1~2초 결제 API 지연 극복)
- **측정**: DB 예매 건수 정확히 1,000건, 타임아웃 응답 최소화
