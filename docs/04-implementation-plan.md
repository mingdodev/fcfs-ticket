# 실험 단계 계획 및 기술 리스크

## 4.1 실험 단계 계획

| 단계 | 목표 | 주요 활동 |
|---|---|---|
| Phase 1 | 기능 구현 검증 | 단일 서버 배포, 정합성 시나리오 A 통과 |
| Phase 2 | 단일 서버 최적화 | DB Lock 전략, 커넥션 풀, 스레드 풀 튜닝 → 목표 TPS 달성 |
| Phase 3 | 인프라 스케일업 (20만 크레딧) | 서버 사양 업그레이드 or 캐시/큐 도입, 비용 대비 성능 비교 |

## 4.2 핵심 기술 리스크

| 리스크 | 설명 | 대응 전략 (예시) |
|---|---|---|
| Race Condition | 동시 `remainingTickets` 조회 시 Oversell 발생 |  |
| Thread Starvation | 결제 API 1~2초 대기로 스레드 풀 소진 | 비동기 처리, 스레드 풀 크기 조정 |
| DB Connection Pool 고갈 | 대량 동시 요청으로 커넥션 대기 | HikariCP 튜닝, 쿼리 최소화 |
| Cascading Timeout | 결제 지연이 전체 응답 지연으로 전파 | 결제 API 타임아웃 + 서킷 브레이커 |
